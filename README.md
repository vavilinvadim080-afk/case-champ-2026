# Кейс-Чемпионат 2026 — лендинг + личный кабинет

Лендинг кейс-чемпионата с регистрацией, входом и загрузкой PDF-решений.
Данные хранятся в SQLite, файлы — на диске сервера.

## Что под капотом

- **Фронт** — один `index.html`, без сборщиков и фреймворков
- **Бэк** — Node.js + Express, `~350` строк в одном `server.js`
- **БД** — `node:sqlite` (встроен в Node 22, без нативных модулей)
- **Загрузка файлов** — `multer` с лимитом размера и фильтром PDF
- **Пароли** — `scrypt` + соль через `node:crypto`
- **Сессии** — random-токены в таблице `sessions`, по 30 дней

Минимум зависимостей. Всего два пакета ставится через npm: `express` и `multer`. Никакого webpack, vite, react, orm.

## Как запустить

Нужен **Node.js 22.5 или новее** (для встроенного `node:sqlite`).
Проверить: `node -v`. Если младше — поставь с [nodejs.org](https://nodejs.org) или через nvm.

```bash
cd "Лендинг Кейс-Чемпинат"
npm install
npm start
```

Откроется на <http://localhost:3000>. Всё.

Если порт занят: `PORT=4000 npm start`.
Поменять лимит файла: `MAX_FILE_MB=50 npm start`.
Поменять папку с данными: `DATA_DIR=/path/to/data npm start`.

### Режим разработки (hot-reload)

```bash
npm run dev
```

Node перезапускает сервер при правках `server.js` (флаг `--watch`).

## Где хранятся данные

После первого запуска создаётся папка `data/`:

```
data/
  app.db                    ← SQLite-база (пользователи, сессии, метаданные решений)
  uploads/
    1776802430383_test.pdf  ← сами PDF-файлы с префиксом timestamp
```

`data/` и `node_modules/` уже в `.gitignore` (создал) — в git не попадут.

Открыть БД вручную: `sqlite3 data/app.db` (если установлен `sqlite3` CLI) или любой GUI типа DB Browser for SQLite.

Таблицы:
- `users` — email (PK), name, university, team, pass_hash, pass_salt, created_at
- `submissions` — id, team, user_email, title, notes, file_name, file_path, file_size, status, created_at
- `sessions` — token (PK), user_email, expires_at

## API (для справки)

| Метод | Путь | Что делает |
|---|---|---|
| `GET`  | `/api/config` | Конфиг для клиента (лимит файла) |
| `POST` | `/api/register` | Регистрация. Body: `{email, name, university, team, password}` |
| `POST` | `/api/login` | Вход. Body: `{email, password}` |
| `POST` | `/api/logout` | Выход (Bearer token) |
| `GET`  | `/api/me` | Профиль + статус команды (Bearer) |
| `POST` | `/api/submission` | Загрузка решения (multipart: team, title, notes, file). Bearer |
| `GET`  | `/api/submissions` | Список решений своей команды (Bearer) |
| `GET`  | `/api/submission/:id/download` | Скачать PDF (Bearer, только своей команде) |
| `GET`  | `/api/health` | Проверка жив ли сервер |

## Как это работает с точки зрения пользователя

1. Заходит на сайт → нажимает **«Личный кабинет»** или **«Создать кабинет»**.
2. Регистрируется: имя, email, вуз, команда, пароль. Команду можно указать позже.
3. В ЛК видит статус команды: «Решение не загружено».
4. Нажимает большую кнопку **«Загрузить решение»** → разворачивается форма.
5. Заполняет название решения, аннотацию, тянет PDF в dropzone.
6. Клиент сразу проверяет: только PDF, не тяжелее 25 МБ (конфигурируется). Показывает размер.
7. Сервер проверяет повторно (multer-лимит + MIME), сохраняет файл на диск, пишет запись в БД.
8. После отправки бейдж меняется на «Решение принято», появляется «Скачать текущее» и «Заменить решение».
9. История всех отправок внизу — первая помечена как «Текущее», остальные видны как версии.
10. При закрытии вкладки и возврате через неделю — авто-логин по токену в localStorage.

Один участник от команды сдаёт — другие участники заходят под своей учёткой, видят **тот же статус команды** (решения привязаны к `team`, а не к user). Любой может заменить текущее решение.

## Ограничения и безопасность

- **Лимит файла**: по дефолту 25 МБ. Проверяется на клиенте (сразу) и на сервере (жёстко через multer). Меняется через env `MAX_FILE_MB`.
- **MIME**: `application/pdf` либо `.pdf` расширение. Остальное отклоняется.
- **Авторизация**: Bearer-токен в заголовке `Authorization`. Хранится в localStorage.
- **Пароли**: scrypt-хэш + случайная 16-байтная соль. Ни сами пароли, ни токены в логи не пишутся.
- **Доступ к файлам**: скачать PDF может только участник той же команды. Проверка по `users.team == submissions.team`.
- **Что не сделано боевого грейда** (для кейс-чемпа на прод это не нужно, но имей в виду):
  - Токен в localStorage теоретически достаётся через XSS — для серьёзного продакшена надо перейти на httpOnly cookie + CSRF-токен
  - Нет rate-limiting — при публичном деплое добавь `express-rate-limit`
  - Нет подтверждения email — поставь, если нужен антиспам (nodemailer + ссылка)
  - Нет HTTPS — обязателен для прода, ставь за nginx/Caddy с автоматическим SSL

## Как деплоить

### Свой VPS (самый прямой путь)

```bash
# На сервере:
git clone <твой репо>
cd case-champ
npm install --omit=dev
pm2 start "node --experimental-sqlite --no-warnings server.js" --name case-champ
pm2 save && pm2 startup
```

Перед pm2 — поставить nginx reverse-proxy на порт 3000 с HTTPS (Let's Encrypt через certbot или сразу Caddy, у которого SSL из коробки).

### Бесплатные хостинги

- **Railway** — коннект репо, задать `PORT` и старт-команду `npm start`. Диск персистентный.
- **Render** — то же самое, Free tier с 512 МБ.
- **Fly.io** — `flyctl launch` → конфиг → `fly deploy`. Понадобится volume для `data/`.

Важно: у бесплатных PaaS бывают эпhemeral-диски — папка `data/` может чиститься. Тогда используй внешнее хранилище:
- PostgreSQL от платформы (вместо SQLite)
- S3-совместимое хранилище (Backblaze B2, Cloudflare R2) для файлов

### Docker

Не сделал Dockerfile, потому что деплой в один Node-процесс через pm2 проще. Если нужно — добавлю.

## Структура проекта

```
case-champ-2026/
├── public/              ← статика, отдаваемая HTTP (всё, что должно быть публичным)
│   ├── index.html       ← фронт (HTML+CSS+JS, всё в одном файле)
│   └── gpn-energy.svg   ← логотип Газпром нефти
├── server.js            ← Express API + SQLite — НЕ отдаётся через HTTP
├── package.json         ← зависимости и npm start
├── package-lock.json
├── README.md            ← этот файл
├── case-example.pdf     ← образец кейса (НЕ публичный)
├── .gitignore           ← data/, node_modules/, .env
├── data/                ← создаётся при запуске, gitignore + не отдаётся через HTTP
│   ├── app.db           ← SQLite с пользователями и сессиями
│   └── uploads/         ← загруженные PDF
└── node_modules/        ← после npm install
```

Важно: только `public/` доступно через HTTP. Корень проекта (`server.js`,
`package.json`, `data/`, `.env`) полностью изолирован от веба.

## Что проверено

- ✓ Регистрация + дубль email → 409
- ✓ Вход + неверный пароль → 401
- ✓ /api/me возвращает профиль и статус команды
- ✓ Загрузка PDF работает, файл скачивается идентичным
- ✓ Лимит 25 МБ → 413 с понятным сообщением
- ✓ Не-PDF → 415
- ✓ Без авторизации → 401
- ✓ Скачивание чужого решения (другая команда) → 403
- ✓ Несуществующий id → 404
- ✓ Persistence: после рестарта сервера пользователи и решения на месте
- ✓ Клиент: HTML-структура валидна, JS без синтаксических ошибок, адаптив от 320 px

## Кастомизация

- **Цвета**: `:root` в `<style>` — замени `--accent`, `--accent-2`, `--accent-3`
- **Тексты**: ищи по якорям `#about`, `#case`, `#stages`, `#prizes`, `#faq`
- **Лимит файла**: env `MAX_FILE_MB` либо правка константы в `server.js`
- **Сессия**: `SESSION_TTL_MS` в `server.js` (по умолчанию 30 дней)
- **Логотип**: 2 места (nav + footer), блок `.logo` — подставь `<img>` вместо букв «КЧ»
- **Жюри**: замени инициалы `.juror__avatar` на `<img>` с фото
- **Призы / даты / партнёры**: чистый HTML, просто текстом
